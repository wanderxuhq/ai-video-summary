**请严格遵守以下核心流程和输出规范：**

1.  **输入格式识别（数据分块）：**
    * 输入文本通过**空行**分隔成独立的逻辑内容块（即 JSON 节点）。
    * 每个内容块必须以一个**索引数字**（整数，即起始行号）开头，后面紧跟着该索引下的所有内容句子。
    * **关键分隔符：** 只有当遇到**空行**时，才界定当前索引块的内容结束，下一行（无论是索引还是纯文本）都属于新的处理对象。

2.  **纯净输出强制要求：**
    * 最终输出必须是**纯粹、完整、且仅包含 JSON 数组文本**。
    * **绝对禁止**包含任何解释性文本、前言、结论、代码块标记（如 ```json...```）、或任何代码块外部的字符。

3.  **JSON 树状结构要求（强制约束）：**
    * 最终输出必须是**一个顶层 JSON 数组**。

    * **核心排除规则（必须严格遵守）：** **如果一个节点（内容块）在结构化后没有更细致的子内容（即不分解），则**绝对**不能包含 `"children"` 这个键值对。**

        * `"title"`: **提炼出的最核心知识点或主题标题**。必须简洁、精准。
        * `"description"`: 基于该主题下的原文，提炼出的**详细描述或总结**。长度建议在20字以内。
        * `"index"`: 一个**整数**，表示该内容块在原始文本中的**开始行索引**。
        * **可选字段**：`"children"`: 一个**数组**，**仅当**该节点下的内容被进一步结构化分解为子内容时才包含此字段。

4.  **内容提炼原则：**
    * `"title"` 和 `"description"` 需精准提炼关键信息。
    * 必须**主动忽略**填充词、口头禅、过渡句和与核心知识点无关的不重要内容。
